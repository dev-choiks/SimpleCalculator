# (C# 코딩) 심플 사칙연산기 (SimpleCalculator)

## 개요
- C# 프로그래밍 학습.
- 1줄 소개: 사용자로부터 두 개의 수치와 연산자를 입력받아 사칙연산을 수행하고, 그 결과를 전체 수식과 최종 결과값이라는 두 가지 방식으로 화면에 출력하는 심플 사칙연산기 프로그램입니다 [2, 3].
- 사용한 플랫폼:
  - C#, .NET Windows Forms, Visual Studio, GitHub.
- 사용한 컨트롤:
  - Label, TextBox, Button.
- 사용한 기술과 구현한 기능:
  - Visual Studio 폼 디자이너를 활용하여 윈도우 기본 계산기와 유사한 직관적인 GUI(Graphic User Interface)를 기획하고 구성했습니다 [3]. 각 컨트롤의 이름은 txtInput, btnPlus, btnEqual 등 용도를 명확히 알 수 있도록 명명 규칙을 철저히 준수하여 가독성을 높였습니다.
  - int.Parse() 메서드를 적극적으로 사용하여 TextBox를 통해 입력받은 문자열(string) 데이터를 실제 산술 연산이 가능한 32비트 정수형(int) 타입으로 강제 변환하는 데이터 형 변환(Type Casting) 기술을 적용했습니다.
  - C#의 핵심 사칙연산 기호인 덧셈(+), 뺄셈(-), 곱셈(*), 나눗셈(/) 연산자를 활용하여 실제 수리적 계산을 수행하는 데이터 처리 로직을 구현했습니다.
  - 모든 연산이 완료된 결과 데이터는 ToString() 메서드를 통해 다시 문자열로 가공했습니다 [6]. 여기에 문자열 보간법을 결합하여, 전체 수식의 흐름까지 화면에 명확하게 출력하는 입력-처리-출력의 완벽한 데이터 사이클을 설계했습니다.

- 과제 내용
- 
  - Label(텍스트 표시), TextBox(수식 및 결과 표시), Button(숫자 패드 및 연산자) 컨트롤을 폼 화면에 적절히 배치하고 속성을 설정합니다.
  - 0~9 숫자 버튼을 클릭했을 때 메인 TextBox 영역에 해당 숫자가 텍스트 형태로 누적되어 입력되도록 구현합니다.
  - 입력된 문자열을 int 형으로 바꾸어 덧셈(+) 연산을 수행한 뒤, 최종 결과를 다시 문자열로 변환하여 출력하는 데이터 흐름을 구현합니다.
  - 계산 내용 전체가 5 x 2 = 10과 같이 수식 형태로 모두 표시되는 방식과, 피연산자가 하나씩 표시되다가 최종 결과값만 표시되는 2가지 출력 방법을 동시에 구현하여 사용자 편의성을 높입니다.

- 구현 내용과 기능 설명
- 
  - Windows 폼 상단에 연산의 전체 과정을 기록할 보조 출력창인 txtOutput을 두고, 그 바로 아래에 현재 누른 숫자와 최종 결과값을 직관적으로 띄워줄 메인 입력창인 txtInput을 배치하여 시각적인 데이터 흐름을 완벽히 분리했습니다. 
  - 0부터 9까지의 숫자 버튼들에 클릭(Click) 이벤트를 연결했습니다. 이벤트 발생 시 sender as Button을 이용해 클릭된 특정 버튼의 텍스트 값(Text)을 가져온 뒤, 메인 txtInput 창에 문자열 결합 연산자(+=)로 연속해서 이어 붙여지도록 코딩했습니다. 초기값이 0일 때는 Clear()로 지우고 숫자가 입력되게 하는 예외 처리도 고려했습니다.
  - 사용자가 덧셈(+) 버튼(btnPlus)을 클릭하면 메인 입력창의 텍스트를 int.Parse()를 이용해 숫자로 변환하여 전역 변수(num1)에 보관하도록 했습니다. 동시에 상단 보조 창에는 문자열 보간법을 사용해 첫 번째 피연산자와 덧셈 기호를 함께 출력하여 입력 과정을 표시하고, 다음 숫자 입력을 위해 메인 창을 비웠습니다.
  - 최종적으로 계산(=) 버튼(btnEqual) 클릭 시 두 번째 피연산자(num2)까지 받아와 덧셈 산술 연산을 수행합니다. 계산된 정수형 결과값(result)은 ToString()을 통해 다시 문자로 가공하여 하단 txtInput 창에 깔끔하게 띄우고 상단에는 전체 수식을 출력했습니다. 이를 통해 프로그램 내에서 입력(문자)에서 변환 및 처리(숫자)를 거쳐 출력(문자)으로 이어지는 데이터 사이클을 완벽하게 구현해 냈습니다.

- 과제 내용
- 
  - 기존 덧셈 기능에 더해 뺄셈(-), 곱셈(*), 나눗셈(/) 버튼을 추가하여 사칙연산 기능을 완벽히 구성합니다.
  - 각 사칙연산 버튼 클릭 시 연산자 기호만 적절하게 변경하여 과제1과 동일한 계산 로직 및 데이터 흐름이 적용되도록 구현합니다.

- 구현 내용과 기능 설명
- 
  - 기존 더하기 버튼의 레이아웃을 바탕으로 뺄셈(btnMinu), 곱셈(btnMulti), 나눗셈(btnDivi) 기능을 담당할 Button 컨트롤들을 폼 화면 우측에 추가로 나란히 배치하여 실제 계산기의 구색을 온전히 갖추었습니다.
  - 새롭게 추가된 각 사칙연산 버튼에 Click 이벤트를 개별적으로 연결했습니다. 연산 버튼을 누를 때마다 입력창(txtInput)에 있는 첫 번째 피연산자 문자열을 int로 변환해 임시 전역 변수에 저장합니다. 그 직후 보조 창(txtOutput)에 해당 연산 기호(-, *, /)가 덧붙여진 전체 수식이 시각적으로 나타나도록 코드를 작성하고, 메인 창은 Clear() 메서드를 호출해 깨끗하게 비워 두 번째 숫자를 받을 준비를 마쳤습니다.
  - 최종 계산을 수행하는 btnEqual의 클릭 이벤트 로직 내부에는 사용자가 마지막으로 누른 연산자가 무엇인지 기억하고 판별하는 조건문 기반의 분기 처리를 구현했습니다. 이를 통해 사용자가 선택한 연산자에 맞춰 알맞은 산술 연산(+, -, *, /)이 정확히 분기되어 실행되도록 기능을 대폭 고도화했습니다. 결과적으로 덧셈뿐만 아니라 다양한 사칙연산 수식을 정상적으로 연산하여 사용자 화면에 2가지 방식(전체 수식 표시, 최종 결과값 표시)으로 정확히 출력해 내는 데 성공했습니다.
